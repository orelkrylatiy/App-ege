# Глоссарий проекта

Документ описывает ключевые сущности предметной области. Используется как
основа для проектирования модели данных, API и UI. Все названия в коде и
интерфейсе должны соответствовать терминам из этого документа.

## Решения по архитектуре данных

- **Без отдельной сущности User.** Tutor и Student — независимые сущности
  с собственными учётными данными. В MVP это проще, нет лишних связей.
  При необходимости в будущем можно вынести общие поля в User.
- **File — отдельная сущность.** Файлы используются и в Assignment
  (материалы от репетитора), и в Submission (ответ ученика). Отдельная
  сущность даёт единую логику работы с файлами и метаданными.
- **В MVP один Student → один Tutor.** Many-to-many откладывается.

## Сущности

| Сущность | Описание |
|----------|----------|
| [Tutor](./entities/Tutor.md) | Репетитор — ведёт учеников, создаёт и проверяет ДЗ |
| [Student](./entities/Student.md) | Ученик — получает ДЗ от репетитора и сдаёт ответы |
| [Assignment](./entities/Assignment.md) | Домашнее задание, выданное ученику |
| [Submission](./entities/Submission.md) | Ответ ученика на ДЗ |
| [Invite](./entities/Invite.md) | Приглашение для регистрации ученика |
| [File](./entities/File.md) | Метаданные прикреплённого файла |
