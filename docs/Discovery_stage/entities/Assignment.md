# Assignment

**Определение:** Домашнее задание, которое репетитор выдаёт конкретному
ученику. Содержит описание задачи, материалы и дедлайн. Имеет статус,
отражающий стадию выполнения.

## Атрибуты

- `id: string` — уникальный идентификатор
- `title: string` — название
- `description: string` — описание задачи
- `deadline: Date` — срок сдачи
- `status: AssignmentStatus` — текущий статус (см. ниже)
- `tutorId: string` — FK на [Tutor](./Tutor.md) (автор)
- `studentId: string` — FK на [Student](./Student.md) (получатель)
- `createdAt: Date` — дата создания
- `updatedAt: Date` — дата последнего изменения

## Возможные статусы (`AssignmentStatus`)

- `IN_PROGRESS` — В работе (выдано, ученик ещё не сдал)
- `UNDER_REVIEW` — На проверке (ученик отправил ответ)
- `DONE` — Сделано (репетитор принял)

> Если `deadline` уже прошёл, а статус всё ещё `IN_PROGRESS` — UI помечает
> такое ДЗ как **просроченное** (красная подсветка). Отдельным статусом
> не выделяется, это вычисляемое свойство.

## Связи

- → [Tutor](./Tutor.md): **many-to-one** (много ДЗ принадлежат одному репетитору)
- → [Student](./Student.md): **many-to-one** (много ДЗ назначены одному ученику)
- → [Submission](./Submission.md): **one-to-one** (одно ДЗ → один ответ; пересдача в MVP не предусмотрена)
- → [File](./File.md): **one-to-many** (к ДЗ может быть прикреплено несколько файлов от репетитора)

## Заметки

- Связь со Student — many-to-one, НЕ one-to-one (у ученика много ДЗ).
- В v2 добавятся: статус `NEEDS_REVISION` (на доработку), комментарии, оценка.
