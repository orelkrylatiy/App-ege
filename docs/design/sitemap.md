# Карта экранов

Документ описывает все экраны приложения, их URL и ключевые переходы.
Используется как основа для проектирования роутинга, навигации и wireframes.

## Решения по UX

- **Сценарий приглашения ученика:** репетитор сначала создаёт карточку
  ученика (ФИО, класс, предмет), затем генерирует для неё ссылку-приглашение.
  Ученик регистрируется по ссылке, привязываясь к уже существующей карточке.
- **Профиль текущего пользователя** доступен по URL без id (`/profile`),
  потому что это всегда «я сам».
- **Список «моих ДЗ»** для ученика — без его id в URL (`/assignments`),
  по той же причине.
- **Действия** (удалить, сменить статус, скопировать ссылку) — не отдельные
  страницы, а кнопки + модалки/PATCH-запросы.

## Public (без логина)

- **Лендинг** `/`
  - Промо, кнопки «Войти» и «Регистрация репетитора»

- **Логин** `/login`
- **Регистрация репетитора** `/signup`
- **Регистрация ученика по приглашению** `/invite/:token`
  - Открывается по ссылке от репетитора, привязывает учётку к существующей карточке Student
- **Восстановление пароля** `/forgot-password`
- **Ошибка / 404** `*`

## Tutor (роль: репетитор, требует логина)

- **Dashboard (главная)** `/`
  - Сводка: ДЗ, требующие проверки + быстрые действия
  - Переходы: → `/students`, → `/assignments/:id` (по клику на ДЗ из сводки)

- **Список учеников** `/students`
  - [действие] Создать ученика → `/students/new`
  - Переход: клик по ученику → `/students/:id`

- **Создание ученика** `/students/new`
  - Форма: ФИО, класс, предмет, контакт, заметки
  - После сохранения → `/students/:id` (карточка нового ученика)

- **Карточка ученика** `/students/:id`
  - Информация об ученике + список его ДЗ
  - [действие] Редактировать → `/students/:id/edit`
  - [действие] Удалить → модалка подтверждения → после удаления `/students`
  - [действие] Пригласить → модалка с ссылкой `/invite/:token`
  - [действие] Создать ДЗ → `/students/:id/assignments/new`
  - Переход: клик по ДЗ → `/assignments/:id`

- **Редактирование ученика** `/students/:id/edit`
  - После сохранения → `/students/:id`

- **Создание ДЗ** `/students/:id/assignments/new`
  - Форма: название, описание, дедлайн, файлы
  - После сохранения → `/assignments/:id`

- **Просмотр / проверка ДЗ** `/assignments/:id`
  - Описание ДЗ, прикреплённые файлы, ответ ученика (если сдан)
  - [действие] Редактировать ДЗ → `/assignments/:id/edit`
  - [действие] Удалить ДЗ → модалка подтверждения
  - [действие] Принять работу (статус → DONE) → PATCH-запрос
  - [действие] Вернуть на доработку (статус → IN_PROGRESS) → PATCH-запрос

- **Редактирование ДЗ** `/assignments/:id/edit`
  - После сохранения → `/assignments/:id`

- **Мой профиль** `/profile`
  - Просмотр своих данных
  - [действие] Редактировать → `/profile/edit`
  - [действие] Выйти → разлогин → `/login`

- **Редактирование профиля** `/profile/edit`

## Student (роль: ученик, требует логина)

- **Dashboard (главная)** `/`
  - Сводка: активные ДЗ + статистика (сколько сделано / в работе)
  - Переход: клик по ДЗ → `/assignments/:id`

- **Список моих ДЗ** `/assignments`
  - Все ДЗ с фильтрами по статусу (активные / на проверке / сделанные)
  - Переход: клик по ДЗ → `/assignments/:id`

- **Карточка ДЗ** `/assignments/:id`
  - Описание, файлы от репетитора, дедлайн, статус
  - Если статус IN_PROGRESS — форма отправки ответа (текст + файлы)
  - Если ответ уже отправлен — отображается ответ и его статус

- **Мой профиль** `/profile`
  - Просмотр своих данных
  - [действие] Редактировать → `/profile/edit`
  - [действие] Выйти → разлогин → `/login`

- **Редактирование профиля** `/profile/edit`

---

## Сквозные элементы (не отдельные экраны)

- **Сайдбар / шапка с навигацией** — на всех страницах после логина.
  Содержимое зависит от роли (у Tutor: Дашборд / Ученики / Профиль;
  у Student: Дашборд / Мои ДЗ / Профиль).
- **Модалка приглашения** — генерация и копирование ссылки `/invite/:token`.
  Открывается с карточки ученика и (опционально) после создания ученика.
- **Модалка подтверждения удаления** — для ученика, ДЗ.
- **Toast-уведомления** — об успехе/ошибке действий.
- **Loading / Empty states** — на списках (нет учеников, нет ДЗ).

| `/students/:id/edit`            | Tutor                | Редактирование ученика                 |
